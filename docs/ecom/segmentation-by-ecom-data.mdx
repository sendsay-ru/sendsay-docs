# Как настроить сегменты по данным продаж

С Модулем **Продажи** вы можете сегментировать подписчиков по тем данным, которые передаются с сайта в систему:

- версия Модуля «Продажи Старт» позволяет создавать сегменты на основе клиентских данных только за последние 48 часов. Для условий недоступны события: Поиск, Добавление в избранное, Предварительный заказ, Товар появился, Изменение цены,
- с полной версией Модуля «Продажи» вы можете создавать сегменты без ограничений.

  [Сравнение версий Модуля Продажи](https://docs.sendsay.ru/ecom/how-to-connect-ecom/#возможности-версий)

:::tip Важно

Настроить сегменты можно только по клиентским событиям и данным, которые передаются с сайта в Sendsay. Например, если настроена передача события **Просмотр товара**, но в событии передаются лишь id товаров без их производителей, то сегментировать аудиторию по просмотренным товарам конкретного производителя не получится.

[Как настроить передачу данных с сайта](https://docs.sendsay.ru/ecom/how-to-configure-data-transfer/#настройка-модуля)

:::

Чтобы настроить условия для сегментов по данным продаж, перейдите в раздел [**Подписчики → Сегменты**](https://app.sendsay.ru/subscribers/segments) и выберите пункт **Продажи** в качестве условия.

![How to create a segment](/img/ecom/segmentation-by-ecom-data/how-to-create-segment.gif)

[Как создать сегмент ](https://docs.sendsay.ru/subscribers/lists-and-segments/what-is-segment/#как-создать-сегмент)

## Фильтр условий клиентских событий

В настройках сегмента можно уточнить условия клиентских событий с помощью кнопки с изображением фильтра:

![Event filter](/img/ecom/segmentation-by-ecom-data/event-filter.png)

- **Количество** — определяет количество действий, совершённых посетителем сайта. Например, следующее условие будет говорить о том, что в сегмент попадут те подписчики, которые оформляли заказ больше трёх раз:

  ![Over three orders](/img/ecom/segmentation-by-ecom-data/over-three-orders.png)

- **Время** — дата события. Например, если событие произошло сегодня или за какой-то конкретный период:

  ![Event runtime](/img/ecom/segmentation-by-ecom-data/event-runtime.png)

- **Атрибуция** — позволяет отследить события, которые произошли после взаимодействия получателя с рассылкой. Здесь есть четыре группы выпусков:

  ![Attribution](/img/ecom/segmentation-by-ecom-data/attribution.png)

  - `<любой выпуск>` — подписчики попадут в сегмент, если совершат указанное событие после перехода на сайт из любой рассылки, отправленной в любой или конкретный период,
  - `<любой выпуск по группе>` — подписчики попадут в сегмент, если совершат указанное событие после перехода на сайт из любого выпуска, отправленного по определённому сегменту в любой или конкретный период,
  - `<любой выпуск по шаблону>` — подписчики попадут в сегмент, если совершат указанное событие после перехода на сайт из выпуска, отправленного по конкретному шаблону в любой или конкретный период,
  - `название выпуска` — подписчики попадут в сегмент, если совершат указанное событие после перехода на сайт из конкретного отправленного выпуска.

- **Товары** — фильтр отбирает товар по его параметрам, передаваемым в Sendsay. Например, при следующих выставленных условиях в сегмент попадут те подписчики, которые просматривали на сайте рюкзаки:

  ![Product name](/img/ecom/segmentation-by-ecom-data/product-name.png)

  Если же в качестве условия добавить второй параметр — например, цену, — то в сегмент попадут подписчики, которые просматривали рюкзаки от 5000 рублей:

  ![Product name and price](/img/ecom/segmentation-by-ecom-data/product-name-and-price.png)

- **Свойства** — фильтрует значения дополнительных полей, передаваемых с сайта в Sendsay. У событий **Сделал заказ** и **Обновлял корзину** в качестве фильтра можно также выбрать данные транзакции — например, её ID или сумму заказа:

  ![Order amount](/img/ecom/segmentation-by-ecom-data/order-amount.png)

  В этот сегмент попадут те подписчики, которые сделали заказ на сумму 10000 рублей или больше.

  При выборе условия **Статус транзакции** необходимо использовать ID статусов, описанных в [таблице статусов заказов](https://docs.sendsay.ru/ecom/how-to-configure-data-transfer/#статусы-заказа). Например, если в сегмент должны попасть подписчики, оплатившие заказ, условия будут выглядеть так:

  ![Transaction status](/img/ecom/segmentation-by-ecom-data/transaction-status.png)

Настраивая сегмент, можно выбрать один или несколько фильтров для уточнения условий. Например, при следующих выставленных условиях в сегмент попадут подписчики, которые не меньше двух раз просматривали рюкзаки стоимостью от 5000 рублей за последние 7 дней:

![Several filters](/img/ecom/segmentation-by-ecom-data/several-filters.png)

## Примеры условий для сегментов по данным продаж

### Сегмент подписчиков, которые оформили заказ после перехода из конкретного письма

Например, необходимо выделить в сегмент подписчиков, кто приобрёл телевизор, перейдя на сайт из конкретного выпуска, — чтобы отправить по этому сегменту письмо с сопутствующими товарами. Тогда условия будут такими:

```
Продажи → Сделал заказ
Атрибуция → Название отправленного выпуска
Товары → Название продукта → содержит → Телевизор
```

![Example](/img/ecom/segmentation-by-ecom-data/example1.png)

Если необходимо выделить подписчиков, которые после перехода из рассылки купили телевизоры стоимостью, например, больше 20000 рублей, к условиям товара необходимо добавить:

```
Цена → больше или равно → 20000
```

![Example](/img/ecom/segmentation-by-ecom-data/example2.png)

### Сегмент подписчиков, которые просматривали товары конкретного производителя

Например, вы хотите отправить проморассылку со скидкой на мебель производителя «Белый город» по тем подписчикам, кто просматривал товары конкретно этого производителя. Условия сегмента будут такими:

```
Продажи → Просматривал товар
Товары → Производитель → равно → Белый город
```

![Example](/img/ecom/segmentation-by-ecom-data/example3.png)

В случае если необходимо выделить подписчиков, которые просматривали товары этого производителя за конкретный промежуток времени, к условиям необходимо добавить фильтр **Время**:

```
Время → за 7 дней
```

![Example](/img/ecom/segmentation-by-ecom-data/example4.png)

### Сегмент подписчиков, которые наиболее часто просматривали товары конкретной категории

Например, необходимо выделить наиболее заинтересованных подписчиков, которые несколько раз просматривали категорию «Беспроводные наушники», — чтобы оповестить их об акции на эту категорию товаров. Тогда условия для сегмента будут такими:

```
Продажи → Просматривал категорию
Количество → больше → 2 раз
Товары → Категория → равно → Беспроводные наушники
```

![Example](/img/ecom/segmentation-by-ecom-data/example5.png)

### Сегмент подписчиков, которые сделали минимум X заказов за последний месяц

Например, необходимо выделить наиболее лояльных клиентов — и рассказать им в рассылке о том, что они стали участниками программы лояльности. Условия для такого сегмента:

```
Продажи → Сделал заказ
Количество → не менее → X раз
Время → за 30 дней
```

![Example](/img/ecom/segmentation-by-ecom-data/example6.png)

Если необходимо, чтобы каждый из этих заказов был на сумму не менее N рублей, необходимо добавить условие:

```
Свойства → Сумма → больше или равно → N
```

![Example](/img/ecom/segmentation-by-ecom-data/example7.png)

### Сегмент подписчиков, которые положили в корзину товар стоимостью больше определённой суммы

Например, необходимо выделить в сегмент тех подписчиков, кто положил в корзину дорогостоящий товар и ушёл. Таких пользователей можно подтолкнуть к покупке, предложив им скидку, бесплатную доставку или другой бонус. Условия для такого сегмента:

```
Продажи → Обновлял корзину
Товары → Цена → больше или равно → N
```

![Example](/img/ecom/segmentation-by-ecom-data/example8.png)

В случае если необходимо конкретизировать товар, можно добавить в условия его название или другие параметры:

```
Товары → Название продукта → содержит → название
```

![Example](/img/ecom/segmentation-by-ecom-data/example9.png)

Или, например, период, в течение которого пользователи откладывали в корзину такой товар:

```
Время → в период → диапазон даты и времени
```

![Example](/img/ecom/segmentation-by-ecom-data/example10.png)

### Сегмент подписчиков, которые купили конкретный товар месяц назад

Например, необходимо выделить в сегмент пользователей, которые совершили покупку месяц назад — с целью напомнить им о том, что пора пополнить запасы. Условия для такого сегмента:

```
Продажи → Сделал заказ
Товар → Название продукта → содержит → название
Время → за интервал → от Y до W дней назад
```

![Example](/img/ecom/segmentation-by-ecom-data/example11.png)
